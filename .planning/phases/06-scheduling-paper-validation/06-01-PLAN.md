# Plan 06-01: Cron Automation, Live Gate Upgrade, Status Monitoring

## Objective

Build unattended cron scheduling (heartbeat/10min, gated cycles/30min, daily forced scan/2AM UTC), upgrade enable_live.py with calibration health + win rate checks, and add status monitoring script.

## Context

- @polymarket-trader/scripts/run_cycle.sh -- existing cycle launcher with --force, PID lock
- @polymarket-trader/scripts/heartbeat.py -- writes state/signal.json
- @polymarket-trader/tools/enable_live.py -- current gate checks cycles + P&L only
- @polymarket-trader/lib/calibration.py -- needs get_calibration_health() function
- @polymarket-trader/lib/db.py -- DataStore with get_paper_cycle_stats()
- @polymarket-trader/tools/setup_schedule.py -- existing single-entry cron setup (to be superseded)

## Tasks

### Task 1: install_cron.sh and status.sh scripts
type="auto"

Build `scripts/install_cron.sh` -- idempotent script that installs 3 crontab entries:
- `*/10 * * * *` heartbeat.py
- `*/30 * * * *` run_cycle.sh (gated by heartbeat)
- `0 2 * * *` run_cycle.sh --force (daily forced scan)

Uses marker comment `# polymarket-trader` for idempotent add/remove.
Auto-detects and removes old `polymarket-agent` cron entries.
Supports `--remove` flag to strip all entries.

Build `scripts/status.sh` -- quick health check showing last heartbeat time, last cycle time, total cycle count, errors in last 24h.

**Done criteria:**
- install_cron.sh creates 3 crontab entries with correct schedules
- install_cron.sh --remove strips all entries
- install_cron.sh is idempotent (running twice produces same result)
- status.sh reads state files and run_cycle.log for summary

### Task 2: Calibration health + live gate upgrade
type="auto"

Add `get_calibration_health()` to `lib/calibration.py` that returns per-category bias data for the live gate.

Upgrade `tools/enable_live.py` with:
- `--check` flag: read-only gate check (no confirmation prompt, no gate pass writing)
- 4 criteria: cycles >= 10, P&L > 0, win rate > 50%, no category calibration bias > -20pp
- JSON output to stdout, human summary to stderr
- Any single criterion failure blocks

**Done criteria:**
- get_calibration_health() returns structured health report from DB
- enable_live.py --check runs all 4 criteria
- JSON output to stdout, human summary to stderr
- Any single failure blocks with clear message

### Task 3: Tests for all new functionality
type="auto"

Write tests for:
- install_cron.sh entry generation (test the crontab building logic)
- get_calibration_health() with various DB states
- enable_live.py --check with pass/fail scenarios for all 4 criteria
- status.sh output parsing

**Done criteria:**
- All new tests pass
- All existing 278 tests still pass
- Test coverage for each new function
