<objective>
Build the calibration tracking library and CLI tool. This is the core learning feedback loop -- when trades resolve, Claude records outcomes, computes Brier scores, tracks per-category accuracy, and auto-generates calibration corrections that feed back into future analysis.

Purpose: Without calibration tracking, the agent cannot learn from its prediction accuracy. This module connects trade resolutions to the calibration-check skill, closing the learning loop.
Output: `lib/calibration.py` library, `tools/record_outcome.py` CLI, `calibration_records` table in DB, comprehensive tests.
</objective>

    **Step 3: Create lib/calibration.py with these functions:**

    `compute_brier_score(stated_prob: float, outcome: str) -> float`:
    - actual = 1.0 if outcome == "WIN" else 0.0
    - return (stated_prob - actual) ** 2

    `interpret_error(error_pp: float, n_trades: int) -> str`:
    - Port from polymarket_claude/scripts/calibration.py reference
    - if n_trades < 5: return "INSUFFICIENT DATA (< 5 trades)"
    - if error_pp > 20: return "UNDERCONFIDENT -- can size up slightly"
    - elif error_pp > 10: return "Slightly underconfident"
    - elif error_pp >= -10: return "WELL CALIBRATED"
    - elif error_pp >= -20: return "Slightly OVERCONFIDENT -- require +6pp edge, -25% size"
    - elif error_pp >= -30: return "OVERCONFIDENT -- require +8pp edge, -50% size"
    - else: return "SEVERELY OVERCONFIDENT -- require +10pp edge, max 1% bankroll"

    `record_calibration_outcome(store: DataStore, market_id: str, category: str, stated_prob: float, outcome: str, pnl: float = 0.0, notes: str = "") -> dict`:
    - Compute brier_score via compute_brier_score
    - actual = 1 if outcome == "WIN" else 0
    - error_pp = (stated_prob - actual) * 100 (signed: positive = underconfident, negative = overconfident)
    - Insert into calibration_records table via store.conn.execute
    - Return dict: {"brier_score": float, "error_pp": float, "category": category, "market_id": market_id}

    `get_calibration_summary(store: DataStore) -> dict`:
    - Query all rows from calibration_records
    - Compute per-category: trades count, wins count, avg brier, avg error_pp, win_rate, total_pnl
    - Compute overall: total trades, overall avg brier, overall avg error_pp, overall win_rate
    - Return dict structured like calibration.json schema

    `generate_corrections(store: DataStore) -> list[dict]`:
    - For each category with >= 5 trades, compute avg error_pp
    - If abs(error_pp) > 10, generate correction:
      - direction: "subtract" if error_pp < -10 (overconfident), "add" if error_pp > 10 (underconfident)
      - amount: min(abs(error_pp) / 100 * 0.7, 0.20) — conservative 70% of observed error, capped at 20pp
      - reason: f"{interpret_error(error_pp, n)} over {n} trades (avg stated {avg_stated:.1f}%, actual win rate {win_rate:.1f}%)"
      - data_points: n
      - expires_after_n_trades: 10
    - Return list of correction dicts

    `regenerate_calibration_json(store: DataStore, json_path: str = "knowledge/calibration.json") -> None`:
    - Per D-02: Called after each record_outcome to keep JSON fresh
    - summary = get_calibration_summary(store)
    - corrections = generate_corrections(store)
    - Build calibration.json structure matching the seed schema:
      - schema_version: "1.0"
      - last_updated: current ISO timestamp
      - summary: {total_trades, overall_brier, overall_error_pp}
      - categories: per-category {trades, brier, error_pp, win_rate, total_pnl} (all 6 categories always present, nulls for empty)
      - corrections: {description: "...", active: corrections_list}
    - Write to json_path with indent=2

    All functions use `from lib.logging_setup import get_logger` with `log = get_logger("calibration")`.
