```python
REQUIRED_SECTIONS = [
    "## Summary",
    "## Markets Analyzed",      # or "## Market Discovery" (existing reports use this)
    "## Trades Executed",
    "## Portfolio State",       # or "## Position Monitor" (existing reports use this)
    "## Resolutions",
    "## Lessons",               # or "## Lessons Learned"
    "## Strategy Suggestions",  # or "## Strategy"
    "## Cycle Metrics",         # or metrics table in Summary
]
```

Be flexible: existing reports use slightly different section names. Accept variants:
- "Markets Analyzed" OR "Market Discovery"
- "Portfolio State" OR "Position Monitor"
- "Lessons" OR "Lessons Learned"
- "Strategy Suggestions" OR "Strategy Changes"
- "Cycle Metrics" OR metrics in Summary table

Knowledge base reference checks:
- `golden_rules`: "golden" (case-insensitive) appears in report
- `calibration`: "calibration" or "brier" (case-insensitive) appears in report
- `playbook`: required only if trades occurred (detect via "Trades executed | 0" or "no trades" pattern). Check for category names: crypto, politics, sports, commodities, entertainment, finance

Strategy drift check:
- Parse "Strategy changes" from metrics table: regex `Strategy changes\s*\|\s*(\d+)`
- If not found, default to 0
- Drift = count > 3

DB checks:
- If trades occurred: query `SELECT COUNT(*) FROM trades WHERE timestamp LIKE '{cycle_date}%'`
- If resolutions occurred: query `SELECT COUNT(*) FROM calibration_outcomes WHERE timestamp LIKE '{cycle_date}%'`
- Use DataStore from lib.db if available, fall back to direct sqlite3 if DB doesn't exist
