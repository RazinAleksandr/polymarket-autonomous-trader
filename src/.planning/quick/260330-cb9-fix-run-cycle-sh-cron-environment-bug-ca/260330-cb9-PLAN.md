
Wait -- `send-keys` does NOT inherit the parent script's env either since tmux starts a fresh shell. Instead, use `tmux new-session` with explicit env var injection. The correct approach:

Replace lines 60-63 with:
```bash
CLAUDE_LOG="$SCRIPT_DIR/logs/claude-${SESSION_NAME}.log"

# Write a temporary runner script that sources the env and runs claude
RUNNER="/tmp/polymarket-cycle-runner-$$.sh"
cat > "$RUNNER" << RUNNER_EOF
#!/usr/bin/env bash
set -euo pipefail

# Source the saved environment
if [ -f "$SCRIPT_DIR/.cron-env" ]; then
    set -a
    source "$SCRIPT_DIR/.cron-env"
    set +a
fi

cd "$SCRIPT_DIR"
source .venv/bin/activate

echo "\$(date -u +%Y-%m-%dT%H:%M:%SZ) Claude starting..." >> "$CLAUDE_LOG"
if claude --dangerously-skip-permissions -p 'run a trading cycle' --verbose 2>&1 | tee -a "$CLAUDE_LOG"; then
    echo "\$(date -u +%Y-%m-%dT%H:%M:%SZ) Claude exited successfully" >> "$CLAUDE_LOG"
else
    echo "\$(date -u +%Y-%m-%dT%H:%M:%SZ) Claude exited with code: \$?" >> "$CLAUDE_LOG"
fi

rm -f "$RUNNER"
RUNNER_EOF
chmod +x "$RUNNER"

tmux new-session -d -s "$SESSION_NAME" "$RUNNER"
```

This runner script approach guarantees the tmux session loads .cron-env, and captures both success and failure exit status to the log file.
